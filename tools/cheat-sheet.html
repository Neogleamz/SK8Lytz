🌙 Session Control

/hello Start every session. Kicks off worktree scan, board health, friction ledger review, and memory load. Always first.
/wind-down End the session. Also triggered by: "good night", "wrapping up", "done for the day"
/panic-button Emergency triage. Something is on fire. Read-only assessment + safe escape route.
Cold start? If you open a new chat and type anything before /hello, the agent auto-detects the cold start and runs the hello protocol anyway.

✨ Magic Words — Exact Phrases

proceed After Quinn generates a plan and you've reviewed it. Hands off to Sage to start coding.
Ship the evolution Approves a Rule Evolution Proposal. The AI updates its own rules.
Hold, here's why... Rejects a Rule Evolution Proposal. Add your reason after the ellipsis.
COWBOY MODE ACTIVATED Bypasses the sprint intercept gate. Use when you want off-sprint work done knowingly.
Never needed: "remember when I told you..." = automatic Friction Event + system fix.

⚙️ The Full Task Lifecycle — What Happens After /start-task

  • P0
    🕵️ Reyes — Source of Truth Prime Reads SESSION_LOG + Master Reference + Protocol Bible. Outputs a Knowledge State. Automatic.
  • P1
    📋 Casey — Sprint Gate Confirms task is in sprint, creates worktree, confirms branch isolation. Automatic.
  • P2
    🏛️ Morgan — Brainstorm Presents design approach, raises failure scenarios, checks rejection register. You say: "what's next" or ask for a plan when ready
  • P3
    📐 Quinn — Plan Generates step-by-step plan artifact. HALTS. Waits for your review. You say: "proceed" to execute, or give feedback to revise
  • P4
    ⚒️ Sage — Code Reads every line before touching it. Implements exactly the plan. Automatic.
  • P5
    🔬 Blake — QA (4-step pipeline) smoke-test → isolated-test → diff-review → qa-tester. Automatic.
  • P5.5
    📋 Avery — Docs Gate Greps the diff. Updates Master Reference §3/§4 if needed. Automatic.
  • P6
    🚀 Taylor — Release npm run verify → commits → gatekeeper merge → Discord notification. Automatic.

⚡ The Self-Evolution System

How the AI evolves itself:
Friction observed →
Filed to Ledger →
3 strikes →
Proposal auto-generated →
You: "Ship the evolution" →
Rule updated ✅
The AI files its own friction events. You'll only see a proposal when a pattern has been observed 3 times. At that point, one word from you bakes the fix into the rules permanently.
Your job: Just notice if something felt wrong. You don't have to remember it — the AI tracks it. To manually trigger: "File a friction event for [thing that went wrong]."

✅ Drift Signals — What to Watch For

Normal Every response starts with [🎯 Jordan | hello | session | cold]. If you don't see a badge header, something's wrong.
Research Reyes always says: "Checking what we already know..." before investigating. If skipped → drift.
Bug River always outputs 3 theories before touching code. Jump straight to fixing → wrong.
Plan Quinn always HALTS and waits for proceed. Code starts without your word → wrong.
Red flag Anonymous response with no persona badge. Say: "Who is speaking right now?" → system resets.

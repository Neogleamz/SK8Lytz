This document outlines how the static image assets extracted from the Zengge APK map to specific Hardware Protocol Patterns, Modes, and interface themes within the SK8Lytz ecosystem.

## 2. Symphony Pixel Screen Modes
These sequences map to the "Symphony Screen" UI, which allows controllers with 2D LED matrix topologies to select intricate pixel art patterns.

**Project Path:** `src/assets/images/zengge_patterns/screen/`

| Sequence Range                | Hardware Behavior                             |
|:------------------------------|:----------------------------------------------|
| `screen0.png` - `screen23.png`| Sequentially maps to Matrix pattern IDs 0-23. Used as graphical shortcuts for pixel-art effects (e.g. scrolling banners, tetris-like overlays). |
| `screenNew16.png`, `17.png`   | Extended / App-updated versions of patterns 16 & 17. |

## 3. Thematic Scenes & Environmental Presets
Zengge utilizes high-resolution `.jpg` photography as thumbnails for their global environmental scene buttons. Selecting these thumbnails sends custom RGB profiles overriding the current state.

**Project Path:** `src/assets/images/zengge_patterns/presets/`

| Base Image Asset | Internal Scene Theme | Protocol Shift |
|:-----------------|:---------------------|:---------------|
| `focus.jpg`      | "Focus"              | Static White / High CCT (`0x31`) |
| `night.jpg`      | "Night"              | Dimmed / Warm Glow (`0x31`) |
| `listen.jpg`     | "Audio Reactive"     | Jump to Music Mode (`0x73`) |
| `meditation.jpg` | "Meditate"           | Slow breath / gradient (`0x51` or `0x42` Zen Mode) |
| `preset_1.jpg` -> `preset_44.jpg` | Generic User Presets | Custom RGB/RBM configurations saved by users. |

---

## 4. Programmatic/Code-Based Modes (NO IMAGES)
It is important to understand that the majority of patterns do **not** have a static image file associated with them in the APK.

- **The 210 RBM Array (0x42)**: Patterns such as "7 Color Jump", "Red Flow", or "White Marquee" do not have dedicated `.png` files. The Zengge UI utilizes standard Dart Canvas commands to dynamically draw miniature colorful visualizers (gradients and solid color boxes) representing these sequences.
- **Fixed & DIY Modes (0x59, 0x51)**: Custom colors, comet, pinch, or solid arrays are configured completely through the color wheel interface.

HealthSyncService.ts attempts to write ExerciseSession, TotalCaloriesBurned, and Distance records to Health Connect, but app.config.js only declares READ permissions (READ_HEART_RATE, READ_STEPS, READ_ACTIVE_CALORIES_BURNED). This will cause a SecurityException at runtime on Android 14+ when the app attempts to sync session data. (Suggested: Add the following write permissions to the Android permissions array in app.config.js:\n- \"android.permission.health.WRITE_EXERCISE\"\n- \"android.permission.health.WRITE_TOTAL_CALORIES_BURNED\"\n- \"android.permission.health.WRITE_DISTANCE\")

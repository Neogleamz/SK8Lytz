# Fix Critical Dependency Vulnerabilities

This plan addresses the remaining security vulnerabilities identified by `npm audit`, specifically targeting the `xmldom` critical injection flaw and `xml2js` moderate prototype pollution.

## User Review Required

> [!WARNING]
> We currently use `@react-native-voice/voice@3.2.4`, which relies on a very old, unpatched version of `@expo/config-plugins` (2.0.4). Running `npm audit fix --force` would violently downgrade our voice library to `3.1.5`, potentially breaking our Voice Command Engine.
>
> To avoid this, we will use modern `npm overrides` to surgically replace the vulnerable, deprecated `xmldom` package deep inside the dependency tree with its safe, renamed successor (`@xmldom/xmldom`), without touching the top-level Voice library.

## Proposed Changes

### Configuration Updates

#### [MODIFY] package.json
- Inject an `overrides` block to strictly enforce safe dependency resolution across the entire tree:
  ```json
  "overrides": {
    "xmldom": "npm:@xmldom/xmldom@^0.8.12",
    "xml2js": "^0.5.0"
  }
  ```

#### [MODIFY] package-lock.json
- Will be automatically regenerated by running `npm install` after the `package.json` updates, locking in the patched sub-dependencies.

## Open Questions

None currently. The approach is isolated and preserves all existing functional code while satisfying the security scanners.

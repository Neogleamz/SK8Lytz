## Proposed Changes

We will modify `.agents/workflows/deepdive-docs.md` to incorporate a new synthesis phase that targets all high-level documentation.

### 1. Enhance Cartographer Directives (Phase 1)
Update the Base Sub-Agent Directive to require Cartographer nodes to explicitly flag architectural surface changes using new metadata tags:
- `[IMPACTS_USER_JOURNEY]` — Flagged when UI flows or user interactions change.
- `[IMPACTS_C4_CONTEXT]` — Flagged when new external systems, databases, or major modules are added.
- `[IMPACTS_STATE_CHART]` — Flagged when XState machines, BLE flows, or hardware lifecycles change.

### 2. Add Phase 4 — High-Level Documentation Synthesis
After the Master Reference is updated in Phase 3, Avery will execute Phase 4:
- Read the flags generated by the 21 sub-agents.
- If `[IMPACTS_USER_JOURNEY]` is flagged, Avery will parse the changes and run a `replace_file_content` update on `tools/User_Journey_Maps.md`.
- If `[IMPACTS_C4_CONTEXT]` is flagged, Avery will update the Mermaid diagrams in `tools/System_Context_Diagram.md`.
- If `[IMPACTS_STATE_CHART]` is flagged, Avery will update the state machines in `tools/State_Charts_UX.md`.
- Avery will scan `tools/SESSION_LOG.md` for new `[DECISION]` entries and automatically sync them into `tools/Architecture_Decision_Records.md`.

### 3. Update Output Messaging
Update the final workflow success message to report not just Master Reference updates, but exactly which high-level documents were synced and modified.
